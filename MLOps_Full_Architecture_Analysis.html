MLOps 3 - Advanced Platform PLANNED

KServe - Advanced Model Serving

Thay thế/bổ sung FastAPI serving hiện tại bằng KServe:

  • InferenceService: Standard model serving protocol
  • Kafka Integration: Consume events từ Kafka → inference → produce results
  • Inference Graph: Chain multiple models (face detect → face recognize → metadata enrich)
  • Canary/A-B Testing: Istio-based traffic splitting
  • Auto-scaling: Scale to zero + KEDA event-driven scaling

Ref: kserve.github.io/website/latest/modelserving/kafka/kafka/

RayServe - Parallel Inference

Chạy song song với KServe cho high-throughput scenarios:

  • Parallel inference: Multiple model replicas
  • Dynamic batching: Batch requests for GPU efficiency
  • Serve DAG: Complex inference graphs
  • Ray Tune: Distributed hyperparameter tuning (tham khảo Bùi Minh Đức)

KEDA - Event-Driven Autoscaling

  • Scale KServe pods based on Kafka queue depth
  • Scale Spark/Flink jobs based on data volume
  • Scale RAG pipeline based on request count
  • Custom metrics from Prometheus

Kubeflow Pipelines

ML pipeline orchestration trên Kubernetes:

  • Training Pipeline: Data prep → Validate (GE) → Train → Evaluate → Register
  • Data Pipeline: Ingest → Transform → Validate → Store
  • GE Component: Great Expectations validation trong pipeline
  • MLflow Integration: Experiment tracking + model registry

Istio Service Mesh

  • Traffic management (canary, blue-green, A/B)
  • mTLS between services
  • Circuit breaker + retry policies
  • Observability (Kiali dashboard)
  • Rate limiting per service

Keycloak Authentication

  • SSO cho tất cả dashboards (Grafana, Airflow, MLflow, DataHub...)
  • API key management cho external access
  • Role-based access control (RBAC)
  • OAuth2 + OIDC integration với Istio

Iter8 - ML Model Testing

Tích hợp Iter8 với KServe để A/B test và progressive delivery:

  • SLO validation (latency, error rate)
  • Automated canary analysis
  • Traffic shifting based on metrics

Ref: iter8.tools/0.9/tutorials/integrations/kserve/

Các Tool RAG và Vai Trò

RAGFlow

Open-source RAG engine với deep document understanding:

  • Document parsing (PDF, images, structured data)
  • Intelligent chunking với multiple templates
  • Visual chunking cho face detection reports
  • Built-in evaluation pipeline
  • Có thể dùng như main RAG orchestrator

Typesense

Search engine cho metadata face detection:

  • Instant search trên metadata (filename, category, face count)
  • Typo-tolerant search
  • Faceted search (filter by date, quality, category)
  • Built-in Natural Language Search (v29+)
  • Microservice riêng, serve nhiều nơi (UI, RAG, API)

Weaviate

Vector database cho semantic search:

  • Store face embeddings + metadata
  • Hybrid search (vector + keyword)
  • Multi-tenant support
  • Tham khảo: deploy-highly-scalable-RAG (Nguyễn Trần Đăng Dương)

Langfuse

LLM observability platform:

  • Trace LLM calls (latency, tokens, cost)
  • Evaluate RAG quality (faithfulness, relevancy)
  • Tích hợp với existing monitoring stack
  • Tham khảo: ChatOpsLLM (Bùi Minh Đức)

Ý Tưởng Microservices Shared

Các tool có thể dùng chung (microservice pattern) cho nhiều pipeline:

  • Kafka: Message bus cho cả data pipeline, ML inference events, và RAG indexing
  • MinIO: Object storage cho raw data, model artifacts, và RAG documents
  • Great Expectations: Validate data ở data pipeline + validate model input ở serving + validate RAG chunks
  • DataHub: Metadata cho data pipeline lineage + model metadata + RAG document catalog
  • Prometheus + Grafana: Unified monitoring cho tất cả services
  • Keycloak: Single auth provider cho tất cả UI dashboards
  • KServe: Serve cả face detection model lẫn LLM/embedding model
